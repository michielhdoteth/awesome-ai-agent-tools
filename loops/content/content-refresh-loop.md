# The Content Refresh Loop

**Category:** Content | **Author:** Matthew Berman
**Published:** 2026-06-21 | **Source:** https://signals.forwardfuture.ai/loop-library/

## Use When

Content has become stale and needs systematic updating with current information.

## Prompt

> Review all content in scope for outdated information, broken links, stale examples, and changed product behavior. For each piece, compare current state against the source of truth. Update material that has drifted, verify examples and links work, and open a pull request with the changes. Stop when every piece of content in scope has been verified or the budget is exhausted.

## Verification

- **Title:** All content in scope is verified against current state.
- **Detail:** Broken links, stale examples, and outdated information are fixed.

## Steps

1. Inventory all content in scope and categorize by type, age, and last review date.
2. Compare each piece against current product behavior, code, and documentation.
3. Update material that has drifted, fix broken links, refresh stale examples.
4. Open a pull request with all verified changes.

## Why

Content freshness directly impacts user trust and search visibility. Systematic review prevents gradual decay.
