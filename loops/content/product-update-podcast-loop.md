# The Product Update Podcast Loop

**Category:** Content | **Author:** Pierson Marks
**Published:** 2026-06-21 | **Source:** https://signals.forwardfuture.ai/loop-library/loops/product-update-podcast-loop/

## Use When

You need to turn daily product changes into a short podcast episode.

## Prompt

> Each night, review publicly released product changes and select only those users need to know. Verify each against the product, docs, or release notes. Use the Jellypod MCP to turn the approved changes into a three-to-five-minute podcast explaining what changed, why it matters, and how to try it. Check the script and audio for accuracy, clarity, and pronunciation. If nothing meaningful shipped, make no episode. Ask before publishing.

## Verification

- **Title:** The episode accurately covers every meaningful public update.
- **Detail:** Review-ready three-to-five-minute episode, or confirmed no-episode result.

## Steps

1. Collect the previous day's public product changes, documentation, and release notes.
2. Select the changes most meaningful to users and verify what actually shipped.
3. Use Jellypod to draft a three-to-five-minute episode.
4. Review the script and audio against the sources, request approval before publishing.

## Why

Fixed release window keeps coverage current. Editorial selection prevents automated reading of commit titles.
