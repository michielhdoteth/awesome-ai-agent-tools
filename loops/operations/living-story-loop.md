# The Living Story Loop

**Category:** Operations | **Author:** Buddy Hadry (@buddyhadry)
**Published:** 2026-06-21 | **Source:** https://signals.forwardfuture.ai/loop-library/loops/living-story-loop/

## Use When

You need a recurring narrative that preserves project meaning and tracks threads across time.

## Prompt

> On each [window], read the configured repositories, goals, prior STORY.md, and optional authorized sources. Update project files, then write STORY.md with focus, deadlines, open threads, and evidence-backed recent wins. Carry every prior thread forward, prove it finished, or mark it STALE/NEEDS-REVIEW—never silently drop one. Archive the snapshot and record the change.

## Verification

- **Title:** The current story accounts for every prior thread and supports every recent win with evidence.
- **Detail:** No thread is silently dropped.

## Steps

1. Read the configured repositories, goals, personal context, optional authorized sources, previous STORY.md.
2. Refresh each project record with current activity, branch state, shipped evidence.
3. Write the new story with interpretation, focus, deadlines, open threads, evidence-backed recent wins.
4. Reconcile every previous thread, archive the verified snapshot.

## Why

Recurring narrative preserves meaning behind activity without letting old commitments disappear.
